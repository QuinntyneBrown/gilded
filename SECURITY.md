# Security Policy

## Supported Versions

Gilded is pre-1.0 and under active development. Security fixes are handled on a
best-effort basis for the latest code on `main`.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Tagged releases | No stable releases yet |

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected vulnerabilities.

Report security issues by email to `quinntynebrown@gmail.com` and include:

- a short summary of the issue and why it matters
- reproduction steps or a proof of concept
- the affected commit SHA, branch, or file paths if known
- any environment details needed to reproduce the problem
- suggested mitigations, if you have them

If you are unsure whether an issue is security-sensitive, report it anyway.

## What to Expect

- Initial acknowledgement target: within 5 business days
- Status updates target: every 10 business days while the report is active
- Coordinated disclosure: after a fix or mitigation is available

Please avoid public disclosure until the issue has been investigated and a
remediation plan is in place.
