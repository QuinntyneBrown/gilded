# L1 - High-Level Requirements

## L1-001: Couple-Oriented User Accounts
The system must allow two spouses to register, authenticate, and link their accounts as a couple so that shared and spouse-only features are scoped to that couple.

## L1-002: Counsellor Discovery by Location
The system must let authenticated users discover Christian counsellors near a given postal code, ordered by proximity.

## L1-003: Web-Sourced Counsellor Ingestion
The system must be able to ingest counsellor records from automated web research and present them as first-class, searchable entries.

## L1-004: Manual Counsellor Submission
The system must allow authenticated users to add counsellors manually when a counsellor is not already present in the directory.

## L1-005: Ratings, Reviews, and Comments
The system must allow users to rate counsellors, leave written reviews, and comment on reviews so other couples can evaluate counsellors.

## L1-006: Counsellor Evaluation and Shortlisting
The system must allow a couple to evaluate candidate counsellors side-by-side and designate one as the chosen counsellor to make an appointment with.

## L1-007: Appointment Intent and Contact
The system must let a couple initiate an appointment with their chosen counsellor by surfacing the counsellor's contact/booking information and recording the couple's decision.

## L1-008: Three-Tier Notes (Private, Spouse-Shared, Public)
The system must let users write notes with one of three visibility levels: private (only the author), spouse-shared (only the author and their linked spouse), and public (visible to all platform users).

## L1-009: Authentication and Account Security
The system must authenticate users securely, protect credentials, and enforce session security across all flows.

## L1-010: Authorization and Data Confidentiality
The system must enforce that confidential and spouse-shared notes are never exposed to unauthorized users, including via API, search, or caching layers.

## L1-011: Mobile-First Responsive Experience
The system must be designed mobile-first and render correctly across XS (<576px), S (>=576px), M (>=768px), L (>=992px), and XL (>=1200px) viewports.

## L1-012: Performance and Scalability
The system must deliver responsive interactions under expected load, with bounded response times on core flows (search, auth, notes).

## L1-013: Input Validation and Abuse Prevention
The system must validate all user input, moderate public content, and prevent spam, injection, and abusive behavior (OWASP Top 10).

## L1-014: Observability and Operability
The system must emit logs, metrics, and errors sufficient for operating the service in production without exposing confidential note content.

## L1-015: Accessibility
The system must meet WCAG 2.1 AA accessibility standards across all user-facing screens.

## L1-016: Angular Material UI Standard
The system must be built on Angular and must use Angular Material as the exclusive source of UI components, typography, color, theme, and style tokens across the entire application.
