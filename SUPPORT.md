# Support

## How to Get Help

Use GitHub Issues for:

- bug reports
- feature requests
- questions about setup or local development
- documentation corrections

When opening an issue, include the command you ran, the expected behavior, the
actual behavior, and any relevant logs or screenshots.

## Before You Open an Issue

- confirm you are working from the latest `main`
- read the setup instructions in [README.md](./README.md)
- check the requirements and task plan in [docs/specs](./docs/specs) and
  [docs/tasks](./docs/tasks)

## Security Issues

Do not use public issues for vulnerabilities. Follow the reporting guidance in
[SECURITY.md](./SECURITY.md).

## Response Expectations

Support is provided on a best-effort basis. There is no guaranteed response
time or service-level agreement.
